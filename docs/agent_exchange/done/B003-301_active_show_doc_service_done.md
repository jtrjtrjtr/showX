## Decisions Made Within Task Scope

### Import path: dist/ not src/

Used `../../../modules/cuelist-core/dist/persistence/showxPackage.js` (compiled output) instead of the `@showx/module-cuelist-core` workspace package. Rationale: `src/main/tsconfig.json` has `rootDir: "src"` (relative to `src/main/`) and cross-package source imports fail with TS6059. The spec said "No cuelist-core module edits," so the `./persistence/showxPackage.js` export could not be added to cuelist-core's `package.json`. The dist JS files exist; a companion `.d.ts` declaration file was hand-written (it would be auto-generated by `tsc --build`).

### uiPanelBridge.test.ts updated

The `registerUiPanelBridge` signature changed from `(config, ipc?)` to `(config, activeShow, ipc?)`. The existing test needed a mock `ActiveShowDoc` injected. Since the spec mandates existing tests pass, this is correct and necessary.

### close() — dirty flush ordering

In `close()`, the pending timer is cancelled first, then `this.dirty` is checked (which may already be false if `flushSave()` ran concurrently). Setting `dirty = false` before `await saveShowxPackage` in `flushSave()` prevents a double-save if `close()` fires during a mid-await `flushSave()`.
