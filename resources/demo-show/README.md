# Demo Show

A theatrical sample show with 25 cues across 3 departments. Designed to be the "start here" experience in ShowX 0.1.

## What's in here

**Show**: Demo Show (Demo Venue, 2026-06-01)
**Mode**: Rehearsal
**Cuelist**: Main Show — 25 cues

### Departments + devices

| Dept | Device | Transport | Address |
|------|--------|-----------|---------|
| LX | ETC Eos (LX console) | OSC | 127.0.0.1:8000 |
| SX | QLab (SX playback) | OSC | 127.0.0.1:53000 |
| VIDEO | disguise (video server) | OSC | 127.0.0.1:9000 |

### Cue structure

| Section | Cues | Departments | Triggers |
|---------|------|-------------|---------|
| Pre-show | Q01–Q05 | LX only | manual |
| Act 1 | Q06–Q15 | LX + SX | manual + auto_follow |
| Act 2 | Q16–Q25 | LX + VIDEO | manual + auto_continue |

**Special cues:**
- **Q11 Storm starts** — compound: LX scene 47 + SX music cue fire together
- **Q14 Battle climax** — group: fires Q10 + Q11 + Q12 in parallel

### Trigger types

- `manual` — SM presses GO
- `auto_follow` — fires immediately when previous cue fires
- `auto_continue` — fires after a configurable delay (500–2000 ms)

### Routing rules

1. `lx_ref` payloads → lx_eos
2. `SX`-tagged payloads → sx_qlab
3. `VIDEO`-tagged payloads → video_disguise
4. Fallback (catch-all) → lx_eos

### Operators

- Stage Manager (SM)
- LX Operator

## How it's bundled

This directory is copied into the `.app` bundle as `Contents/Resources/demo-show/`. It is read-only inside the bundle.

When the user clicks **Open Demo Show**, ShowX copies `demo.showx` to `~/Documents/ShowX/Demo Show.showx` so they can edit freely without modifying the original.

## Source of truth

The `demo.showx` fixture is generated by `scripts/generate-demo-fixture.mjs` from the same data as `src/modules/cuelist-core/src/document/demoFactory.ts`. Both must stay in sync. Run the script after editing either source.
